## 一、关键技术发现（最重要）

### 1.1 闲鱼有个"PC H5 mtop 搜索端点"——不需 sign

来自 `superboyyy/xianyu_spider/spider.py:124`：

```
https://h5api.m.goofish.com/h5/mtop.taobao.idlemtopsearch.pc.search
```

打开 `goofish.com` 搜索时浏览器**自然会触发**这个 mtop 请求并拿到结构化 JSON 响应。**Playwright 只需 `page.on("response", ...)` 拦截即可拿到全部商品**——比我们当前从 DOM 文本里硬抽好太多。

**意义**：我们当前 `crawl_xianyu_pw` 抽到 1 条的根因是 DOM 选择器命中差。换 mtop 拦截可一页拿 30+ 条结构化数据。

### 1.2 淘宝 mtop 调用模板

来自 `TSDK README` 示例：

```
https://h5api.m.taobao.com/h5/<api_name>/<version>/?jsv=2.4.8&appKey=12574478
  &t=<timestamp_ms>&sign=<sign>&api=<api_name>&v=<v>&data=<json_data>
```

签名规则（来自 `tmallSign/routes/tmall.js` 注释 `s = a(token + "&" + i + "&" + r + "&" + data)`）：

```python
sign = md5(f"{token}&{timestamp_ms}&{appKey}&{json_data}").hexdigest()
# token 取自 cookie 中的 _m_h5_tk 字段（格式 "xxx_yyy"，取 _ 前部分）
# appKey 通常是 "12574478"
```

**意义**：淘宝走 `s.taobao.com/search` SSR 路径已被 `rgv587_flag` 硬封。但**淘宝也有同样的 PC H5 mtop 搜索端点**——理论上可以用同样的"Playwright 打开 m 端 + 拦截 response"思路绕过。

**改造成本**：~1.5 小时
**风险**：闲鱼若改 mtop API 路径失效（但比 DOM 稳定）。
**License**：MIT，注明出处即可。

### 2.2 ⚠️ zhinianboke/xianyu-auto-reply（C · 暂不动）

### 2.5 🔧 cclient/tmallSign（B · 抄公式不抄代码）

**核查文件**：`routes/tmall.js`（209 行）、README

**实质**：209 行 JS 实现的 `sign(e)` 函数 = **MD5**（IV 常量 `1732584193, 4023233417...` 是 MD5 标准）。

Python 等价：

```python
import hashlib

def mtop_sign(token: str, t: str, app_key: str, data: str) -> str:
    """淘宝 mtop sign。
    参考：vendor/cclient-tmallSign/routes/tmall.js#L19-L189 (Apache-2.0)
    本质是 md5(token & t & appKey & data)。
    token 取 cookie '_m_h5_tk' 字段中 '_' 前的部分。
    """
    return hashlib.md5(f"{token}&{t}&{app_key}&{data}".encode()).hexdigest()
```

**怎么用**：直接在我们的代码里写这一个函数（带出处注释），**不需要起 Express 子进程、不复用 JS**。
**改造成本**：5 分钟
**风险**：零（MD5 算法本身不会变；如果淘宝改了拼接顺序才会失效）。
