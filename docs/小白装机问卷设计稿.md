# 小白装机问卷设计稿

> 目的：先用固定问题而不是 AI 自由理解，逐步收敛用户需求，确定 `CPU / 显卡方向`、`预算范围`，再输出对应档位的整机配置。

---

## 一、设计目标

- 面向完全不懂硬件的小白用户
- 优先通过问卷而不是开放输入来确定需求
- 前几题先确定 `CPU优先`、`GPU优先`、`均衡型`、`低功耗办公型`
- 后几题再收敛预算、升级预期、是否包含外设
- 最终输出 `低 / 中 / 高` 三档配置建议

---

## 二、设计原则

- 不要一开始就问用户 CPU、显卡型号，小白通常无法回答
- 先问用途，再问负载，再问偏好，最后问预算
- 问题尽量使用单选，便于在微信小程序中直接分支和计分
- 问题数量控制在 `6~9` 题
- 前 `3` 题必须能大致判断 `CPU / 显卡方向`
- 尽量避免过长术语，必要时用通俗描述替代

---

## 三、推荐问卷结构

### 1. 你的电脑主要拿来做什么？

选项：

- 日常办公
- 上网课 / 学习
- 写代码 / 开发
- 平面设计
- 视频剪辑
- 3D 建模 / 渲染
- 打游戏
- 多种用途都要

用途：

- 作为一级分支题
- 先把用户分到办公、学习、开发、设计、游戏、多用途等大类

---

### 2. 下面哪种情况最接近你的实际使用？

选项：

- 同时开很多网页、文档、聊天软件
- 会跑开发环境、数据库、虚拟机、Docker
- 会修图、排版、做海报
- 会剪 1080P / 4K 视频
- 会玩大型 3A 游戏
- 会跑 3D / CAD / 建模渲染
- 只是轻度使用，不卡就行

用途：

- 作为“真实负载题”
- 修正第一题里“用途模糊”或“多种用途都要”的情况

---

### 3. 你更在意哪一项？

选项：

- 反应快、开软件快
- 多开不卡、运行稳定
- 游戏帧数高
- 设计 / 渲染效率高
- 安静省电
- 性价比最高

用途：

- 判断用户偏向 `CPU优先`、`GPU优先`、`均衡型`、`低功耗`

---

### 4. 你会不会玩游戏？如果会，偏向哪类？

选项：

- 基本不玩
- LOL / CF / DOTA2 这类网游
- 原神 / 永劫无间 / 吃鸡
- 黑神话 / 赛博朋克 / 3A 大作
- 主要是主机模拟器 / 多开游戏

用途：

- 判断是否必须独显
- 判断显卡性能下限

---

### 5. 你做设计或内容创作吗？

选项：

- 不做
- Photoshop / Illustrator / Figma 为主
- Premiere / 剪映 / 达芬奇
- CAD / SketchUp / SolidWorks
- Blender / C4D / 3ds Max

用途：

- 补充判断生产力需求
- 避免把设计用户误分到办公机

---

### 6. 你对电脑寿命和升级有什么期待？

选项：

- 能用就行，2~3 年够了
- 希望 3~5 年都够用
- 希望留升级空间
- 一步到位，尽量别后悔

用途：

- 决定是否推荐更大电源、更多内存、更大 SSD、更新平台

---

### 7. 你更偏向哪种预算心态？

选项：

- 越便宜越好，够用就行
- 性价比优先
- 贵一点没关系，但要明显更舒服
- 预算不是第一位，体验优先

用途：

- 先判断用户心理预算层级
- 避免用户直接报一个不准确的金额

---

### 8. 你的整机预算更接近哪一档？

选项：

- 2500 以内
- 2500~4000
- 4000~6000
- 6000~9000
- 9000~12000
- 12000 以上
- 还没想好，先看推荐

用途：

- 最终确定预算范围

---

### 9. 是否需要包含这些外设？

选项：

- 只要主机
- 主机 + 显示器
- 主机 + 显示器 + 键鼠
- 全套都要

用途：

- 防止预算误判
- 识别“总预算是否包含外设”

## 四、推荐分类结果

问卷最终先归类到这 4 种方向：

- `CPU优先`
- `GPU优先`
- `均衡型`
- `低功耗办公型`

---

## 五、建议评分思路

### 1. 可维护的评分字段

- `cpu_score`
- `gpu_score`
- `balance_score`
- `office_score`

### 2. 推荐判断逻辑

- 如果“开发 / 多开 / 虚拟机 / 办公多任务”权重高，则提高 `cpu_score`
- 如果“游戏 / 3D / 渲染 / 剪辑”权重高，则提高 `gpu_score`
- 如果“平面设计 / 轻剪辑 / 多用途”较多，则提高 `balance_score`
- 如果“轻办公 / 学习 / 省电 / 低预算”较多，则提高 `office_score`

### 3. 示例判定规则

- `gpu_score - cpu_score >= 2` → `GPU优先`
- `cpu_score - gpu_score >= 2` → `CPU优先`
- 分数接近且用途较杂 → `均衡型`
- 轻度用途且预算低 → `低功耗办公型`

---

## 六、预算区间建议

推荐不要只给单点预算，而是给区间：

- `低预算`：2500~4500
- `中预算`：4500~7000
- `高预算`：7000~12000
- `高阶生产力 / 发烧`：12000+

如果用户选择“全套都要”，建议从总预算中预留：

- 显示器：`700~1500`
- 键鼠：`100~300`

---

## 七、结果页建议输出

用户答题后，结果页建议固定输出以下 4 块内容：

### 1. 配置方向

示例：

- `你的需求更适合：均衡型偏 CPU`

### 2. 推荐预算范围

示例：

- `建议主机预算 5000~6500，更容易配到够用且有升级空间的方案`

### 3. 三档配置

- 低档：够用版
- 中档：推荐版
- 高档：一步到位版

### 4. 升级建议

示例：

- `如果后续可能剪视频，优先把内存从 16GB 升到 32GB`

---

## 八、适合小程序的数据结构方向

建议拆成两层：

### 1. 问题配置 JSON

- 只存题目、选项、题型、顺序

### 2. 评分规则 JSON

- 只存每个选项对应加到哪类分数
- 以及最终如何映射到 `CPU优先 / GPU优先 / 均衡型 / 低功耗办公型`

这样后续你改题目文案时，不需要改核心逻辑。

---

## 九、后续分析重点

后续评审这套问题是否合适时，建议重点看这几个点：

- 是否有题目重复表达同一件事
- 是否有题目太专业，小白看不懂
- 是否前 3 题真的足以判断 CPU / 显卡方向
- 是否预算问题过早出现，影响用户回答
- 是否还缺少对“便携 / 静音 / 品牌偏好 / 二手接受度”的补充题
- 是否结果能稳定映射到三档配置，而不会出现大量模糊情况

## 十二、方向判定规则（V1）

### 1. 低功耗办公型

满足任一情况优先判定：

- `office_score >= 5` 且 `gpu_score <= 2`
- 用户主要是办公 / 学习 / 轻度使用
- 预算整体较低，且无游戏、剪辑、建模需求

推荐特征：

- 优先核显
- 更关注稳定、省电、低噪音
- 配置重点在够用，不追求独显

---

### 2. GPU优先

满足条件：

- `gpu_score - cpu_score >= 2`

常见来源：

- 3A 游戏
- 3D 建模
- 渲染
- 视频剪辑
- 高强度设计软件

推荐特征：

- 显卡优先定档
- CPU 保证不拖后腿即可
- 内存通常建议从 32GB 起步

---

### 3. CPU优先

满足条件：

- `cpu_score - gpu_score >= 2`

常见来源：

- 开发
- 虚拟机
- Docker
- 多开办公
- 模拟器多开

推荐特征：

- 优先更高核心数 / 更新平台
- 更重视内存容量和 SSD 空间
- 独显不是第一优先级

---

### 4. 均衡型

满足条件：

- `abs(cpu_score - gpu_score) < 2`
- 或者 `balance_score` 明显更高

常见来源：

- 平面设计
- 轻剪辑
- 多用途混合
- 性价比导向

推荐特征：

- CPU 和显卡都不极端偏科
- 更适合输出“低 / 中 / 高”三档方案

---

## 十三、预算档位映射（V1）

### 1. 原始预算档位

可先用 `budget_level` 映射：

- `0` → 超低预算
- `1` → 低预算
- `2` → 中低预算
- `3` → 主流预算
- `4` → 中高预算
- `5` → 高预算

---

### 2. 主机预算修正

如果用户预算包含外设：

- `主机预算 = 总预算 - peripheral_cost`

然后再重新落入主机预算区间：

- `2500 以下`
- `2500~4000`
- `4000~6000`
- `6000~9000`
- `9000~12000`
- `12000 以上`

---

### 3. 升级倾向修正

如果 `upgrade_score >= 2`，则在不超预算太多的前提下优先：

- 内存从 `16GB` 升 `32GB`
- SSD 从 `512GB` 升 `1TB`
- 电源预留更大冗余
- 主板平台不要过旧

---

## 十四、三档配置输出规则（V1）

最终结果页不直接给一个方案，而是统一给三档：

### 1. 低档：够用版

用途：

- 满足当前主要需求
- 价格最低
- 牺牲部分冗余和升级空间

适合：

- 预算紧
- 首次装机
- 追求先上车

---

### 2. 中档：推荐版

用途：

- 性价比最平衡
- 作为默认推荐
- 最适合大多数用户

适合：

- 主流预算
- 想稳定使用 3~5 年

---

### 3. 高档：一步到位版

用途：

- 提前覆盖未来需求
- 更好的静音、散热、扩展、寿命

适合：

- 预算较宽松
- 不想后续频繁升级

---

## 十五、程序落地建议

如果你后续做小程序逻辑，建议按下面三步实现：

### 1. 收集答案

- 每题只记录选项 ID

### 2. 累计分数

- 按本节规则累加 `cpu_score / gpu_score / balance_score / office_score`
- 同时记录 `budget_level / upgrade_score / peripheral_cost`

### 3. 输出结果

依次生成：

- 配置方向
- 推荐主机预算范围
- 低 / 中 / 高三档方案
- 一句升级建议

## 十六、当前规则的明显不足

这套规则现在能用来做第一版，但还有几个明显问题要后续继续修：

- “学习”用户跨度太大，从网课到建模都可能出现
- “开发”用户差异也很大，前端轻开发和 AI / Android 开发不是一个档位
- “设计”需要继续拆成平面、视频、建模三类，否则 GPU 推荐会偏保守
- 预算题目前还是偏粗，后面可能要再加“只买新件 / 接受二手”一题
- 还没有引入品牌偏好、体积需求、静音需求、便携需求
