## 一、功能总览

给完全不懂硬件的小白用户，通过 9 道题（Q1 多选+排优先级，Q2–Q9 单选）收敛出：

1. **配置方向**：CPU优先 / GPU优先 / 均衡型 / 低功耗办公型（四选一）
2. **预算档位**：从 2500 以下到 12000+ 共 6 档
3. **三档硬件方案**：低档（够用）/ 中档（推荐）/ 高档（一步到位）
4. **升级建议**：1~3 条针对性文字提示

所有输出基于固定评分规则，相同输入保证相同输出。

#### Q1 位次权重

每类**内部**按位次衰减：

```
weight[0] = 1.0   (排 #1)
weight[1] = 0.6   (排 #2)
weight[2] = 0.3   (排 #3)
weight[3+] = 0.15 (排 #4 及之后)
```

计分方式：`score_delta = opt.s[k] × weight[position]`。三类之间**独立累加**，互不干扰。

## 三、评分状态

完整状态对象（所有字段初始为 0 或 null）：

```js
S = {
  cpu: 0,             // 数值，累加
  gpu: 0,             // 数值，累加
  balance: 0,         // 数值，累加
  office: 0,          // 数值，累加
  upgrade: 0,         // 0~3，由 Q6 一次性赋值
  budgetHint: null,   // 0~3，由 Q7 一次性赋值
  budgetFloor: null,  // 0~5，由 Q8 一次性赋值（q8g 不赋值）
  peripheral: 0,      // 金额，由 Q9 一次性赋值
  side: []            // 记录 Q1 每一项的加权细节（结果展示用）
}
```

浮点累加后四个主分最终保留 1 位小数：`round(score * 10) / 10`。

## 四、方向判定（按优先级从上往下第一个命中的生效）

```
if S.office >= 5 and S.gpu <= 2   →  "低功耗办公型"
if S.gpu - S.cpu >= 2             →  "GPU优先"
if S.cpu - S.gpu >= 2             →  "CPU优先"
else                              →  "均衡型"
```

阈值 `2` 是刻意留的容差，防止 1 分差就切换方向。如果实际测试样本大量落在"均衡型"，再收紧到 `1.5`。

## 五、预算映射

### 步骤 1 · 确定总预算档位

```
if S.budgetFloor !== null:   level = S.budgetFloor      (第 8 题优先)
elif S.budgetHint !== null:  level = S.budgetHint       (第 7 题兜底)
else:                        level = 2                  (默认中低档)
```

### 步骤 2 · 预算档位表

```
level  label          lo      hi       plan (规划价)
0      2500 以下      0       2499     2200
1      2500~4000      2500    3999     3500
2      4000~6000      4000    5999     5000
3      6000~9000      6000    8999     7500
4      9000~12000     9000    11999    10500
5      12000 以上     12000   99999    14000
```

### 步骤 3 · 扣除外设

```
hostPlan = max(total.plan - S.peripheral, 2200)
```

然后找 `hostPlan` 落在哪个 bucket：`hostLevel = bucket.findIndex(b => hostPlan in [b.lo, b.hi])`。

### 三档派生

基于基础模板 `tpl`（来自 `direction × hostLevel`）：

```
低档 = 低价系数 0.85 × tpl.range，内存/SSD/电源降一档
中档 = 1.00 × tpl.range，内存/SSD/电源与模板一致（若 upgrade ≥ 2 则提一档）
高档 = 1.22 × tpl.range，内存/SSD 提一档，电源提一档
```

内存/SSD/电源 升降规则（字符串替换，按容量顺序找到匹配即替换）：

```
upMem:   8GB→16GB→32GB→64GB
downMem: 64GB→32GB→16GB→8GB
upSsd:   256GB→512GB→1TB→2TB (500GB 视作 512GB)
downSsd: 2TB→1TB→512GB→256GB
upPsu:   300W→450W, 400W→550W, 450W→550W, 500W→650W,
         550W→750W, 650W→850W, 750W→850W, 850W→1000W
```

## 九、算法伪代码

```
function decideAll(answers, sideState):
  # 1. 校验
  missing = []
  if sum(len(sideState[g]) for g in Q1_GROUPS) == 0: missing.add("Q1")
  for q in QUESTIONS:
    if answers[q.id] is None: missing.add(q.id)
  if missing: return {error: ...}

  # 2. 评分
  S = {cpu:0, gpu:0, balance:0, office:0, upgrade:0, budgetHint:null, budgetFloor:null, peripheral:0, side:[]}
  for q in QUESTIONS:
    opt = q.opts[answers[q.id]]
    for key in ["cpu","gpu","balance","office"]: S[key] += opt.s[key] or 0
    for key in ["upgrade","budget","budgetFloor","peripheral"]:
      if opt.s[key] is defined: S[key] = opt.s[key]
  for g in Q1_GROUPS:
    for (i, id) in enumerate(sideState[g.id]):
      w = SIDE_WEIGHTS[i] or SIDE_WEIGHTS[-1]
      opt = g.opts[id]
      for key in ["cpu","gpu","balance","office"]: S[key] += (opt.s[key] or 0) * w
      S.side.append({group:g.title, id, text:opt.text, rank:i+1, weight:w})
  for key in ["cpu","gpu","balance","office"]: S[key] = round(S[key]*10)/10

  # 3. 方向
  if   S.office >= 5 and S.gpu <= 2:   direction = "低功耗办公型"
  elif S.gpu - S.cpu >= 2:             direction = "GPU优先"
  elif S.cpu - S.gpu >= 2:             direction = "CPU优先"
  else:                                direction = "均衡型"

  # 4. 预算
  level = S.budgetFloor ?? S.budgetHint ?? 2
  totalBucket = BUDGET_BUCKETS[level]
  hostPlan = max(totalBucket.plan - S.peripheral, 2200)
  hostLevel = index of bucket where hostPlan in [bucket.lo, bucket.hi]

  # 5. 三档配置
  tpl = CONFIG_TEMPLATES[direction][hostLevel]
  boosted = S.upgrade >= 2
  tiers = [makeTier("低档", tpl, -1, boosted),
           makeTier("中档", tpl,  0, boosted),
           makeTier("高档", tpl, +1, boosted)]

  # 6. 升级建议
  tips = []
  if S.upgrade >= 2:                   tips.add("...")
  if direction=="GPU优先" and S.cpu<=2: tips.add("...")
  # ...

  return {S, direction, totalBucket, hostBucket, hostPlan, tiers, tips}
```

## 十一、实现清单（给新开发者）

1. **数据层**（纯数据）
   - `Q1_GROUPS` + `SIDE_WEIGHTS`（第二节）
   - `QUESTIONS`（第二节 Q2–Q9）
   - `BUDGET_BUCKETS` + `CONFIG_TEMPLATES`（第六节）

2. **评分层**（纯函数，无 UI 依赖）
   - `scoreAnswers(answers, sideState) → S`
   - `decideDirection(S) → {dir, reason}`
   - `decideBudget(S) → {totalBucket, hostBucket, hostPlan, ...}`
   - `decideConfig(direction, hostLevel, S) → tiers[3]`
   - `suggestUpgrade(direction, S) → tips[]`
   - `decideAll(answers, sideState) → {error?, S, dir, budget, tiers, tips}`

3. **UI 层**（渲染 / 事件）
   - Q1：多选池 × 3 类
   - Q2–Q9：单选组
   - 结果卡片：方向 / 预算 / 三档 / 建议
   - 提交按钮、清空按钮、填样例按钮（调第十节五组样例）

## 十二、已知限制

- 硬件价格是 2026-04 估算，需接入爬虫实价后替换 `CONFIG_TEMPLATES`
- 权重曲线 `[1.0, 0.6, 0.3, 0.15]` 和方向阈值 `2` 是经验值，应该用真实埋点数据校准
- "学习 / 开发 / 设计"这些词在设计稿 §十六 已标注跨度大——如果测试样本大量落在"均衡型"且用户不满意，优先考虑在 Q1 池子里细分这几类的子项
- 暂未覆盖：品牌偏好 / 体积静音需求 / 二手接受度 / 便携需求——后续迭代加入
