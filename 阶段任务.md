# 多平台商品爬虫 · 阶段任务清单

> 目标：构建一个**可被外部调用**（Python import / CLI / HTTP API）的统一爬虫，覆盖 **闲鱼、京东、淘宝、拼多多** 四大平台，按关键词或类目搜索商品并归一化输出。
>
> 基本原则：**优先复用 GitHub 上活跃维护的开源项目**，自己只做归一化、调度、错误处理与对外接口层。反爬策略变化快，所有依赖项目必须验证最近 6 个月内仍有更新或已确认可用。

## 一、技术总体设计

```

│   ├── config.yaml.example     # 平台 cookie / 限流 / 代理配置示例

│   ├── models.py               # Product / SearchResult 数据模型
│   ├── base_spider.py          # 爬虫抽象基类 + 重试装饰器
│   └── settings.py             # 配置加载
├── spiders/
│   ├── __init__.py

├── state/                      # 各平台登录态 cookie / storage_state
└── tests/

```

### 1.2 对外调用方式（三种）

1. **Python 模块**：`from spiders.jd import JDSpider; JDSpider().search("RTX 4090", page=1)`
2. **CLI**：`python cli.py search --platform jd --keyword "RTX 4090" --pages 3`
3. **HTTP API**：`POST /search { "platform": "jd", "keyword": "RTX 4090" }`

### 1.3 通用技术栈

| 模块 | 选型 | 备注 |
|---|---|---|
| 日志 | `loguru` | 滚动文件 + 控制台彩色 + 错误单独归档 |
| HTTP | `httpx` (异步) / `requests` | 平台不同 |
| 浏览器自动化 | `playwright` 或 `DrissionPage` | 闲鱼/淘宝必需 |
| 数据模型 | `pydantic v2` | 统一 Product 字段 |
| 重试 | `tenacity` | 指数退避 |
| 配置 | `PyYAML` + `pydantic-settings` | yaml + 环境变量 |
| API | `fastapi` + `uvicorn` | 对外 HTTP 调用 |
| 持久化 | `SQLite` (sqlmodel) | 去重 + 历史价格 |

## 四、错误与日志策略（**Phase 0 必须落地**）

### 4.1 异常分层（`core/exceptions.py`）

```
SpiderError (基类)
├── ConfigError              配置缺失/错误
├── NetworkError             网络/超时/代理问题
├── LoginRequiredError       cookie 失效或未登录
├── AntiSpiderError          触发风控/CAPTCHA/滑块
│   └── RateLimitError       触发限流（HTTP 429 / 业务码）
├── ParseError               页面结构变化导致解析失败
└── PlatformError            平台返回业务错误
```

每个异常携带：`platform`、`url`、`raw_response`（截断到 2KB）、`request_id`，便于日志关联。

### 4.2 日志规范（`core/logger.py`）

- **库**：loguru（开箱即用、好看、性能足够）
- **输出**：
  - 控制台：彩色，INFO 及以上
  - `logs/{platform}_{YYYY-MM-DD}.log`：DEBUG 及以上，按天 rotate，保留 14 天
  - `logs/error.log`：仅 ERROR/CRITICAL，保留 30 天，便于事后排查
- **结构化字段**：每条日志都带 `platform`、`spider`、`keyword`、`request_id`
- **关键事件必记**：
  - 请求发起（DEBUG，含 URL 与 params）
  - 请求完成（INFO，含状态码、耗时）
  - 解析结果（INFO，含数量）
  - 命中反爬 / 登录失效（WARNING，含响应片段）
  - 异常抛出（ERROR，含 traceback）

### 4.3 调试钩子

- 配置开关 `debug.save_html = true` → 出错时把响应 HTML/JSON 落到 `logs/debug/{platform}_{ts}.html`
- 配置开关 `debug.screenshot_on_error = true` → Playwright/DrissionPage 出错时自动截图到 `logs/debug/`
- 全局 `--verbose` 提升日志等级到 DEBUG

## 五、风险与应对

| 风险 | 应对 |
|---|---|
| 平台反爬升级，参考项目失效 | 锁定 `requirements.txt` 版本；每个 spider 写最小 smoke test，CI 每天跑一次 |
| 拼多多 anti_content JS 变化 | 定期从 onetwo1/pinduoduo 同步 JS；解密失败 → `AntiSpiderError` |
| 淘宝/闲鱼登录态过期 | 配置过期阈值，过期前 1 天 WARN；过期后强制走人工补登流程 |
| 法律合规 | 仅个人使用，**严格控制 QPS（默认 0.5 req/s）**，不对外提供商业服务，遵守 robots.txt 中可遵守的部分 |
| 数据准确性（价格促销价 vs 划线价） | Product 模型同时存 `current_price` 与 `origin_price`，双字段记录 |
