## 关键设计决策

### 日志
- 库：**loguru**（开箱即用，handler 配置简单）
- 控制台 INFO+，彩色
- `logs/{date}.log` DEBUG+，按天滚动，保留 14 天
- `logs/error.log` ERROR+，10MB 滚动，保留 30 天
- 上下文绑定字段：`platform / spider / keyword / request_id`
- 调试落盘工具：`dump_debug_artifact()` 保存出错时的 HTML/JSON 到 `logs/debug/`

### 异常分层
```
SpiderError
├── ConfigError
├── NetworkError              （可重试）
├── LoginRequiredError        （不可自动重试，需人工补登）
├── AntiSpiderError           （触发风控）
│   └── RateLimitError        （限流，退避重试）
├── ParseError                （结构变化，需更新代码）
└── PlatformError             （业务错误）
```
所有异常携带 `platform / url / raw_response（截断 2KB）/ request_id`。

### 数据模型
- `Product`：`platform / item_id / title / url / current_price / origin_price / image_url / shop_name / location / sales / is_second_hand / crawled_at / raw`
- `SearchResult`：`platform / keyword / page / total_pages / products[] / extra`
- 去重键：`{platform}:{item_id}`

### 爬虫基类（待写）
- `BaseSpider(ABC)` 抽象方法：`search(keyword, page, **kw) -> SearchResult`、`detail(item_id) -> Product`、`login_state_valid() -> bool`
- 内置：`@retry`（tenacity，指数退避，仅对 NetworkError/RateLimitError 重试）、统一日志注入、限流（`min_interval` 字段）
