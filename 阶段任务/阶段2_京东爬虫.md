## 任务清单

- [ ] 2.1 调研 `https://search.jd.com/Search?keyword=xxx&page=N` 页面结构（lxml 解析）
- [ ] 2.2 调研价格 ajax：`https://p.3.cn/prices/mgets?skuIds=J_xxx,J_yyy`
- [ ] 2.3 编写 `spiders/jd.py`：
  - 类 `JDSpider(BaseSpider)`，platform=Platform.JD
  - 使用 `httpx.Client`（同步即可，京东无强校验）
  - 真实浏览器 UA + Referer + 必要 cookie（thor / __jdv 可选）
- [ ] 2.4 `_parse_list(html)`：xpath 提取每个 `li.gl-item` 的 sku_id、标题、店铺、原价
- [ ] 2.5 `_fetch_prices(sku_ids)`：批量请求价格接口，合并到 product
- [ ] 2.6 `search(keyword, page=1, sort='default')`：拼接 URL，page 实际 `2*page-1`（京东特殊翻页）
- [ ] 2.7 异常映射：
  - HTTP 4xx/5xx → `NetworkError`
  - 跳转到 passport 登录 → `AntiSpiderError`（京东风控少见但有）
  - HTML 找不到 `#J_goodsList` → `ParseError`
- [ ] 2.8 `tests/test_jd.py`：`search("RTX 4090")` 至少返回 30 条，价格非 None

---

## 已知坑

- 京东第 N 页 URL 是 `page=2N-1`（每页含 30 条 + 异步加载 30 条）
- 无 cookie 也能搜，**但价格必须走 p.3.cn**（页面里不直接写）
- 部分商品 sku_id 实际是 mpid（拼购），价格接口需要 `M_xxx` 前缀

---

## 验收标准
- `JDSpider().search("RTX 4090", page=1)` 返回 ≥30 条且 `current_price` 非 None
- 故意传非法 keyword（特殊字符），不崩溃，返回空 result
- 日志中有完整 URL、状态码、解析数量
