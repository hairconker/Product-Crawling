# 阶段 5 · 统一对外接口（CLI + HTTP API）

> 周期：第 13 天
> 目标：把 4 个 spider 包装成**三种调用方式**，外部可零侵入接入。

---

## 任务清单

- [ ] 5.1 `cli.py`：使用 `typer`
  ```bash
  python cli.py search --platform jd --keyword "RTX 4090" --pages 3 --output json
  python cli.py search --platform xianyu --keyword "5080" --price-range 1000-5000
  python cli.py login --platform taobao
  ```
- [ ] 5.2 `api/server.py`：FastAPI
  - `POST /search` body: `{platform, keyword, page=1, max_pages=1, extra={}}` → `SearchResult`
  - `GET /platforms` → 返回支持平台列表与登录状态
  - `GET /health` → 健康检查
- [ ] 5.3 工厂模式：`spiders/factory.py` `def get_spider(platform: Platform) -> BaseSpider`
- [ ] 5.4 统一错误响应：FastAPI 异常处理器把 `SpiderError` 子类映射成 HTTP 状态码
  - `LoginRequiredError` → 401
  - `AntiSpiderError` → 429（含 RateLimitError）
  - `ParseError / PlatformError` → 502
  - `NetworkError` → 503
  - `ConfigError` → 500
- [ ] 5.5 请求日志中间件：每个 HTTP 请求生成 `request_id`，注入 logger
- [ ] 5.6 编写 `Dockerfile` 与 `docker-compose.yml`（API + xvfb，方便服务器部署）

## Python 调用示例（库模式）

```python
from spiders.factory import get_spider
from core.models import Platform

spider = get_spider(Platform.JD)
result = spider.search("RTX 4090", page=1)
for p in result.products:
    print(p.title, p.current_price)
```

## 验收标准

- `curl -X POST localhost:8000/search -d '{"platform":"jd","keyword":"RTX 4090"}'` 返回正常 JSON
- 故意触发各类异常，HTTP 状态码与 body 中 `error_type` 字段正确
- `python cli.py search --platform jd --keyword test` 可直接打印表格结果
