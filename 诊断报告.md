## 二、各平台为什么都失败

### 2.1 京东（根因：当前 IP 被风控）

| 测试 URL | 实际结果 |
|---|---|
| `https://search.jd.com/Search?keyword=i5-12400F` | 302 → `https://www.jd.com/?from=pc_search_sd`（返回首页而非搜索页） |
| `https://www.jd.com/` | 302 → `https://corporate.jd.com/home`（跳到企业站） |
| `https://so.m.jd.com/ware/search.action?keyword=...` | 302 → `https://cfe.m.jd.com/privatedomain/risk_handler/...`（**明确风控页**） |

**结论**：WSL 对外 IP 被京东识别为海外/机房 IP，搜索功能不给返回。
**解决**：需要在国内家庭宽带 IP 下运行，或配置 HTTP 代理（`proxy` 配置项已留好）。

### 2.2 闲鱼 / 淘宝（根因：SPA + 必须登录态）

- `goofish.com/search` 返回的 HTML 只有外壳 JS，商品数据要靠登录后异步 API `mtop.taobao.idle.*` 拉取
- `s.taobao.com/search` 未登录直接跳 `login.taobao.com`

**解决**：这两个平台必须用 Playwright 浏览器 + 导入登录 cookie（见 `阶段任务/阶段1_闲鱼爬虫.md` 和 `阶段3_淘宝爬虫.md`）

### 2.3 拼多多（根因：纯 SSR 空壳，数据 JS 异步渲染）

检查 HTML 中的 window 变量：

```
window.__AD_METRICS_CONFIG__   window.__CDN_FONT__
window.__CMT_HOST__            window.__SSR_STREAM_END__
... 全部是配置/监控，无 rawData 无商品列表
```

**结论**：PDD 已把商品数据改为全 JS 异步加载，不跑浏览器拿不到。
**解决**：需要 `anti_content` JS 解密 + 直接调 API（见 `阶段任务/阶段4_拼多多爬虫.md`）

## 四、要让爬虫真正出结果，按优先级做 3 件事

### 1. 装 pip 和依赖（开发机一次性）

```bash
# 在能联网且有 sudo 的环境下
sudo apt install -y python3-pip python3-venv
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
playwright install chromium
```

### 2. 解决 IP 风控（运行时）

二选一：
- 把脚本部署到国内云服务器 / 国内家庭网络
- 在 `config/config.yaml` 配置 HTTP 代理：

```yaml
proxy:
  enabled: true
  pool:
    - http://user:pass@your-proxy:port
```

### 3. 登录态（闲鱼/淘宝/拼多多）

按各阶段文件导出 cookie：
- 闲鱼：Chrome 扩展导出 `state/xianyu_state.json`（见 [ai-goofish-monitor](https://github.com/Usagi-org/ai-goofish-monitor)）
- 淘宝：`scripts/taobao_login.py` 扫码登录
- 拼多多：浏览器 F12 → Application → Cookies，复制 `PDDAccessToken`
